1. Introduction

Les données manquantes sont omniprésentes en data-mining : questionnaires incomplets, fusion de fichiers, défaillances de saisie, refus de répondre, etc. Leur traitement est indispensable car la plupart des méthodes d'analyse ne sont pas conçues pour les données incomplètes.

La méthode naïve consistant à supprimer les individus incomplets (listwise deletion) n'est pas satisfaisante :

  • Les individus complets ne sont pas nécessairement représentatifs → biais.
  • Le nombre d'individus complets devient vite très petit quand le nombre de variables est grand.
Exemple : si 5 % des valeurs sont manquantes pour chaque variable (MCAR), un jeu de 50 variables contient en moyenne seulement 8 % d'individus complets ($0{,}95^{50} \approx 0{,}08$).

Deux approches :

  1. Pré-traitement : imputation pour se ramener à des données complètes.
  2. Méthodes avancées : algorithmes EM, pondération, qui s'appliquent directement sur les données incomplètes.

2. Mécanismes de manquance

On note $X_{n \times p}$ le tableau de données complet (fictif) et $R_{n \times p}$ la matrice d'indication de manquance : $r_{ij}=1$ si $x_{ij}$ est observé, $0$ sinon. Les méthodes de traitement dépendent du type de mécanisme.

2.1 MCAR (Missing Completely At Random)

$$P(R \mid X) = P(R)$$

La probabilité de manquance est indépendante de toutes les données (observées et manquantes).

  • Les individus complets forment un sous-échantillon représentatif.
  • L'analyse sur les cas complets n'est pas biaisée (mais perte de puissance).
  • Exemple : questionnaire perdu aléatoirement, prélèvement sanguin accidentellement détruit.

2.2 MAR (Missing At Random)

$$P(R \mid X) = P(R \mid X^{\text{obs}})$$

La probabilité de manquance dépend des données observées uniquement.

  • L'analyse par cas complets est biaisée.
  • Hypothèse par défaut de la plupart des méthodes d'imputation.
  • Exemple : les hommes répondent moins à une question sur le poids (la manquance dépend du sexe, qui est observé).
À retenir : L'hypothèse MAR est non testable. Il est recommandé d'inclure des variables auxiliaires (peu/pas de manquantes, sans intérêt scientifique propre) qui aident à expliquer la manquance et rendent MAR plus crédible.

2.3 MNAR (Missing Not At Random)

$$P(R \mid X) = P(R \mid X^{\text{obs}}, X^{\text{mis}})$$

La probabilité de manquance dépend des valeurs manquantes elles-mêmes.

  • Impossible à tester à partir des seules données observées.
  • Nécessite des méthodes spécifiques (pattern mixture models, selection models) ou une analyse de sensibilité.
  • Exemple : les personnes à haut revenu ne déclarent pas leur revenu.

3. Identification des mécanismes

Bien qu'il soit impossible de démontrer le mécanisme, l'analyse exploratoire aide à l'identifier :

  1. Analyse univariée : proportion de manquantes par variable.
  2. Analyse bivariée : lien entre la manquance d'une variable et les valeurs observées d'autres variables (ex. distribution d'une variable selon qu'une autre est observée ou non).
  3. Analyse multivariée : régression logistique de la manquance sur les variables observées.
  4. Test de Little : teste MCAR contre MAR. Compare les profils des individus complets et incomplets.
Limite : Ces analyses ne permettent pas de trancher entre MAR et MNAR (tous deux peuvent produire des différences de distribution). Seule la connaissance du mécanisme de production des données permet de décider.

4. Méthodes par suppression

4.1 Listwise deletion (cas complets)

On supprime toute observation comportant au moins une valeur manquante.

  • Avantage : simplicité, c'est le comportement par défaut de la plupart des logiciels.
  • Inconvénients : perte de puissance ; biais sauf si MCAR ; gaspillage d'information.

4.2 Pairwise deletion

On utilise toutes les observations disponibles pour chaque calcul (moyenne, corrélation).

  • Inconvénient : la matrice de covariance peut ne pas être définie positive.

5. Imputation simple

5.1 Imputation marginale (moyenne / médiane)

Remplace chaque valeur manquante par la moyenne (ou médiane) de la variable. Atténue la variance et réduit les corrélations.

5.2 Imputation par régression

  1. Ajuster un modèle de régression sur les données observées.
  2. Prédire les valeurs manquantes à partir de ce modèle.
  3. Ajouter une perturbation aléatoire (imputation stochastique) pour mieux respecter la distribution.

5.3 Hot-deck ($k$-NN)

Identifier les $k$ individus complets les plus proches (distance euclidienne) et imputer par leur moyenne (quantitative) ou le mode (qualitative).

5.4 Imputation par analyse factorielle

ACP (quantitatives), ACM (qualitatives) ou AFDM (mixtes) via le package missMDA. Particulièrement adapté à la grande dimension, la multicolinéarité et les données mixtes.

5.5 Imputation séquentielle

  1. Ordonner les variables de la moins manquante à la plus manquante.
  2. Initialiser les valeurs manquantes (tirage dans les observées).
  3. Pour chaque variable $X_j$ : ajuster le modèle sur les autres variables, mettre à jour les valeurs imputées.
  4. Répéter l'étape 3 jusqu'à convergence.

5.6 Modèles joints

Spécifier un modèle pour la distribution jointe :

  • Quantitatives : modèle gaussien multivarié (package Amelia).
  • Qualitatives : modèle log-linéaire, classes latentes (package cat).
  • Mixtes : general location model (package mix).
Inconvénient de l'imputation simple : elle ne tient pas compte de l'incertitude d'imputation. Les variances sont sous-estimées, les intervalles de confiance trop courts, et les tests trop libéraux.

6. Imputation multiple — MICE

MICE (Multiple Imputation by Chained Equations) génère $M$ jeux de données imputés (typiquement $M=5$ à $M=20$) en tenant compte de l'incertitude d'imputation.

6.1 Algorithme

  1. Initialisation : imputation provisoire (e.g. moyenne).
  2. Boucle d'imputation : pour chaque variable $X_j$ contenant des valeurs manquantes :
    • Modéliser $X_j$ en fonction de toutes les autres variables (modèle adapté au type), uniquement sur les lignes où $X_j$ est observée.
    • Imputer les valeurs manquantes par tirage dans la distribution prédictive (prédiction + bruit aléatoire).
  3. Cycles : répéter l'étape 2 pendant 5--10 cycles pour stabiliser les imputations (chaîne de Markov).
  4. Répétition : répéter tout le processus $M$ fois → $M$ jeux de données imputés.
  5. Analyse : analyser chaque jeu séparément puis combiner avec les règles de Rubin.

Choix du modèle d'imputation : « Choisir des modèles d'imputation au moins aussi complexes que la méthode qui sera appliquée sur les données imputées » (Schafer, 2003).

Règle pratique : $M$ au moins égal au pourcentage d'observations incomplètes. $M=5$ pour des taux modérés, $M \ge 20$ pour des taux élevés ou des inférences sur des quantiles extrêmes.

7. Règles de Rubin

Soit $\hat{\theta}_m$ l'estimation du paramètre d'intérêt sur le $m$-ième tableau imputé, et $\widehat{\operatorname{Var}}(\hat{\theta}_m)$ sa variance estimée.

7.1 Estimation agrégée

$$\hat{\theta} = \frac{1}{M}\sum_{m=1}^{M} \hat{\theta}_m$$

7.2 Variance totale

$$\widehat{\operatorname{Var}}(\hat{\theta}) = \underbrace{\frac{1}{M}\sum_{m=1}^{M} \widehat{\operatorname{Var}}(\hat{\theta}_m)}_{\text{Variance intra}} + \left(1 + \frac{1}{M}\right) \underbrace{\frac{1}{M-1}\sum_{m=1}^{M} (\hat{\theta}_m - \hat{\theta})^2}_{\text{Variance inter}}$$
  • La variance intra $\bar{U}$ représente l'incertitude dans chaque échantillon.
  • La variance inter $B$ représente l'incertitude due aux données manquantes.
  • Le facteur $(1+1/M)$ corrige l'estimation de $B$ pour un nombre fini d'imputations.

7.3 Intervalles de confiance

L'intervalle de confiance à $95\%$ pour $\theta$ est :

$$\hat{\theta} \pm t_{\nu, 0.975} \sqrt{\widehat{\operatorname{Var}}(\hat{\theta})}$$

où les degrés de liberté $\nu$ sont donnés par l'approximation de Rubin-Schafer.

8. Diagnostic et convergence

8.1 Visualisation des motifs de manquance

  • Matrice des motifs (blanc = observé, rouge = manquant).
  • Heatmap de corrélation entre indicateurs de manquance.

8.2 Convergence de MICE

  • Tracer l'évolution de la moyenne et de l'écart-type des valeurs imputées au fil des cycles.
  • Si la chaîne converge, les fluctuations sont stables (typiquement 5-10 cycles).

8.3 Overimputation

Technique de validation interne : générer des valeurs plausibles également pour les données observées. En générant plusieurs centaines de tableaux, on construit un intervalle de prédiction à 95 % pour chaque valeur observée. Si le modèle est bien choisi, 95 % des valeurs observées sont contenues dans leur intervalle.

9. Autres méthodes

9.1 Algorithmes EM et Data Augmentation

  • EM (Expectation-Maximization) : alterne imputation (espérance) et estimation des paramètres par maximum de vraisemblance.
  • DA (Data Augmentation) : version bayésienne qui alterne génération des données imputées et tirage des paramètres dans leur distribution a posteriori.

Différence avec l'imputation : le modèle d'imputation est le même que le modèle d'analyse. On ne peut pas imputer par $k$-NN et analyser par régression linéaire (contrairement à l'imputation classique).

9.2 Méthodes par pondération

Attribuer un poids aux individus complets, inversement proportionnel à leur probabilité d'être manquants. Issu de la théorie des sondages. Avantage : pas besoin de modéliser la distribution des données. Inconvénient : nécessite de modéliser le mécanisme de manquance.

9.3 Analyse de sensibilité

Puisque MAR n'est pas testable, il est recommandé d'évaluer l'impact d'un écart à MAR :

  • Faire varier $\delta$ dans un modèle où $P(\text{manquant}) = f(X_{\text{obs}}) + \delta \cdot X_{\text{manq}}$.
  • Modèles par patterns de manquance (pattern-mixture models).
  • Analyse de scénarios extrêmes (best-case / worst-case).

# Imputation multiple avec mice library(mice) # Chargement et inspection des motifs data(nhanes) md.pattern(nhanes) # Imputation multiple (m=5, methode PMM) imp <- mice(nhanes, m = 5, method = 'pmm', seed = 123, printFlag = FALSE) # Diagnostics de convergence plot(imp) # Comparaison des distributions observees/imputees densityplot(imp) # Analyse : regression lineaire fit <- with(imp, lm(bmi ~ age + chl)) # Combinaison (regles de Rubin) pooled <- pool(fit) summary(pooled)
# Imputation par forets aleatoires imp_rf <- mice(nhanes, m = 5, method = 'rf', seed = 123, printFlag = FALSE) fit_rf <- with(imp_rf, lm(bmi ~ age + chl)) summary(pool(fit_rf))
# Imputation par analyse factorielle (missMDA) library(missMDA) data(ozone) # Estimation du nombre de composantes nb <- estim_ncpPCA(ozone[, -1], ncp.max = 5) # Imputation par ACP res.impute <- imputePCA(ozone[, -1], ncp = nb$ncp) head(res.impute$completeObs)
# Simulation : impact de l'imputation simple sur les IC set.seed(123) nsim <- 2000 res <- matrix(NA, nsim, 2) for (ii in 1:nsim) { x <- rnorm(500) y <- 2 * x + rnorm(500, sd = 0.5) # Ajout de donnees manquantes MAR prob <- pnorm(1.2 * x - 0.5) ismar <- sapply(prob, FUN = function(p) sample(c(TRUE, FALSE), 1, prob = c(p, 1 - p))) ymar <- y; ymar[ismar] <- NA # Imputation simple par regression stochastique res.lm <- lm(ymar[!ismar] ~ x[!ismar]) ystoch <- ymar ystoch[ismar] <- cbind(1, x[ismar]) %*% res.lm$coefficients + rnorm(sum(ismar), sd = summary(res.lm)$sigma) # Intervalle de confiance se <- sd(ystoch) / sqrt(500) ci_inf <- mean(ystoch) - qt(0.975, 499) * se ci_sup <- mean(ystoch) + qt(0.975, 499) * se res[ii, ] <- c(mean(ystoch), 0 >= ci_inf & 0 <= ci_sup) } cat("Taux de couverture:", mean(res[, 2]), "\n") # Devrait etre < 0.95 (sous-couverture)

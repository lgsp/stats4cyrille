1. Introduction

Cette étude de cas applique l’imputation multiple par équations chaînées (MICE) aux données Diabète (Pima Indians Diabetes Database, 768 observations, 9 variables). L’objectif est de traiter les valeurs manquantes avant une classification supervisée.

Données : https://github.com/niharikagulati/diabetesprediction

1.1 Chargement

library(mice) library(micemd) library(FactoMineR) library(parallel) library(DescTools) library(VIM) library(mvtnorm) library(funModeling) load("diabetes.Rdata") dim(diabetes) # 768 x 9

8 prédicteurs + 1 réponse binaire (Outcome).

2. Analyse exploratoire

2.1 Liens entre variables

summary(diabetes) pairs(diabetes[, 1:8], cex = .2) boxplot(diabetes[,"Age"] ~ diabetes$Outcome)

2.2 Dispositif de manquance

Univarié (aggr) :

res.aggr <- aggr(diabetes) res.aggr$missings cbind(res.aggr$tabcomb, res.aggr$percent)

Bivarié (V de Cramer) :

var.na <- which(res.aggr$missing$Count > 0) pattern <- is.na(diabetes[, var.na]) matcram <- PairApply(pattern, CramerV) PlotCorr(matcram)

Multivarié (ACM) :

MCA(pattern)

2.3 Nature du mécanisme

# Une variable complete (Age) et une incomplete (Skin.Thick) marginmatrix(diabetes[, c("Age", "Skin.Thick")]) # Rouge = valeurs manquantes sur Skin.Thick # Les individus incomplets semblent plus ages → MAR possible # Tous les couples marginmatrix(diabetes[, -ncol(diabetes)], cex = .2, gap = 0) # Matrixplot matrixplot(diabetes, sortby = 8)
À retenir : Si les individus avec valeur manquante sur une variable ont des distributions différentes sur une autre variable complète, le mécanisme est probablement MAR.

3. Imputation multiple avec MICE

3.1 Paramètres par défaut

ParamètreDéfautDescription

maxit5Nombre d’itérations
méthodepmmPredictive Mean Matching (numériques)
res.mice <- mice(diabetes, maxit = 50, printFlag = FALSE) tableau1 <- complete(res.mice, 1)

3.2 Vérification de la convergence

plot(res.mice)

Les courbes doivent se stabiliser (pas de tendance monotone).

3.3 Comparaison des distributions

densityplot(res.mice)

Les distributions imputées (rouge) doivent ressembler aux observées (bleu).

3.4 Analyse par marginmatrix

diabetes.vim <- cbind.data.frame( complete(res.mice), is.na(diabetes)) colnames(diabetes.vim) <- c(colnames(diabetes), paste0(colnames(diabetes), "_imp")) # Couple Age - Skin.Thick marginmatrix(diabetes.vim[, c("Age", "Skin.Thick", "Age_imp", "Skin.Thick_imp")], delimiter = "_imp") # Tous les couples marginmatrix(diabetes.vim, delimiter = "_imp")

Les points imputés (marron/jaune) doivent se situer dans le nuage des points observés.

3.5 Overimputation

L’overimputation traite chaque valeur observée comme manquante, l’impute, et compare à la valeur réelle.

nnodes <- detectCores() - 1 res.mice.over <- mice.par(diabetes, m = 100, nnodes = nnodes, maxit = 20) plotinds <- sample(seq(nrow(diabetes)), size = 30) res.over <- overimpute(res.mice.over, plotinds = plotinds, plotvar = 6, nnodes = nnodes)
Diagnostic : Si les valeurs imputées sont systématiquement éloignées des valeurs observées, le modèle d’imputation est inadapté.

5. Analyse après imputation

5.1 Régression logistique

fit <- with(res.mice, glm(Outcome ~ Preg + Glucose + BP + Skin.Thick + Insulin + BMI + DPF + Age, family = binomial)) length(fit$analyses) # 5 modeles

5.2 Pooling (règles de Rubin)

res.pool <- pool(fit) summary(res.pool)
$\bar{\theta} = \frac{1}{m}\sum_{k=1}^m \hat{\theta}_k$
$T = W + (1 + \frac{1}{m})B$

5.3 Comparaison avec les cas-complets

res.glm.cc <- glm(Outcome ~ ., family = binomial, data = diabetes) summary(res.glm.cc)

Des différences importantes entre les estimations MICE et cas-complets peuvent indiquer un biais de sélection dû aux données manquantes.

6. Analyse de sensibilité

L’analyse de sensibilité par la méthode du delta adjustment teste la robustesse des résultats sous différents scénarios MNAR. On ajoute un décalage $\delta$ aux valeurs imputées de Skin.Thick :

boxplot(diabetes$Skin.Thick) delta <- c(0, 10, 20, 50) imp.all <- vector("list", length(delta)) names(imp.all) <- delta post <- res.mice.over$post for (i in 1:length(delta)){ d <- delta[i] cmd <- paste("imp[[j]][,i] <- imp[[j]][,i] +", d) post["Skin.Thick"] <- cmd imp <- mice(diabetes, post = post, maxit = 5, seed = i, print = FALSE) imp.all[[i]] <- imp } # Comparaison delta=0 vs delta=50 bwplot(imp.all[["0"]]) bwplot(imp.all[["50"]])
Interprétation : Si les conclusions changent peu malgré un décalage important ($\delta = 50$), les résultats sont robustes à l’hypothèse MNAR.

7. Pipeline résumé

  4. Mécanisme → marginmatrix, matrixplot (détection MAR)
  5. Imputation multiple → mice() avec $m$, maxit, méthodes
  6. Diagnostics → convergence (plot), distribution (densityplot), marginmatrix, overimputation
  7. Analyse → with.mids + pool (règles de Rubin)
  8. Comparaison cas-complets → vérification de biais
  9. Sensibilité → delta adjustment pour tester MNAR

1. Contexte

La détection des points atypiques (outliers) est une phase importante de la fouille de données. Ce sont des points qui s'écartent significativement des autres, peu nombreux et situés dans des zones peu denses.

Isolation Forest (Liu, Ting & Zhou, 2008) est une méthode de référence pour la détection non supervisée des anomalies, qui s'appuie sur les forêts aléatoires.

2. Principe

L'approche est non supervisée. Contrairement aux forêts classiques (supervisées), la construction des arbres se fait entièrement au hasard.

Idée centrale : les valeurs atypiques sont isolées rapidement (faible profondeur), car elles se trouvent dans des zones peu denses. Les observations normales nécessitent plus de coupures pour être isolées (grande profondeur).

  1. Construire $B$ arbres aléatoires découpant l'espace.
  2. Pour chaque observation $x$, mesurer la profondeur moyenne $\overline{h}(x)$ dans la forêt.
  3. Calculer un score d'anomalie $s(x,n)$ compris entre 0 et 1.
  4. Seuiller les scores pour identifier les atypiques.
À retenir : Plus un score est proche de 1, plus l'observation est susceptible d'être atypique.

3. Construction des arbres au hasard

3.1 Algorithme

  1. Sélectionner une variable au hasard parmi $p$.
  2. Choisir un point de coupure au hasard dans l'étendue de cette variable.
  3. Segmenter l'échantillon en deux sous-groupes.
  4. Répéter jusqu'à ce qu'un nœud ne contienne qu'un seul individu ou atteigne une profondeur limite.

3.2 Sous-échantillonnage et effet de masque

Pour construire la forêt, on commence par un sous-échantillonnage (tirage aléatoire sans remise). Cela limite l'effet de masque : les valeurs atypiques peuvent elles-mêmes perturber la détection et en masquer d'autres. En tirant un sous-échantillon, les points atypiques sont plus rapidement isolés.

Effet de masque : Un groupe d'anomalies peut se créer une zone dense locale, rendant leur isolement plus difficile. Le sous-échantillonnage réduit ce phénomène.

On répète $B$ fois la procédure (sous-échantillon + arbre) pour obtenir une forêt.

4. Calcul du score

4.1 Profondeur d'isolement

$h(x)$ = profondeur de $x$ dans un arbre (nombre d'arêtes entre la racine et le nœud terminal).

Sur $B$ arbres, profondeur moyenne : $\displaystyle \overline{h}(x) = \frac{1}{B}\sum_{b=1}^{B} h_b(x)$.

4.2 Facteur de normalisation $c(n)$

La profondeur dépend de $n$ : plus il y a d'observations, plus l'arbre sera profond. On normalise par la profondeur moyenne attendue pour un arbre aléatoire :

$c(n) = 2 \times H(n-1) - \dfrac{2(n-1)}{n}$

où $H(k) = \displaystyle\sum_{i=1}^{k} \frac{1}{i}$ est le $k$-ième nombre harmonique.

4.3 Score d'anomalie

$s(x, n) = 2^{-\overline{h}(x) / c(n)}$
Valeur du scoreInterprétation
$s \approx 0.5$Observation normale ($\overline{h}(x) \approx c(n)$)
$s \to 1$Très probablement atypique (isolement rapide)
$s \to 0$Plutôt normale (isolement profond)

5. Paramétrage

5.1 Nombre d'arbres ($B$)

Influe sur la variance de $\overline{h}(x)$. $B = 100$ est souvent suffisant, $B = 300$ donne une très bonne stabilité.

5.2 Taille du sous-échantillon

Liu, Ting & Zhou (2008) recommandent 256 observations. Pas de gain substantiel au-delà. Réduit le coût computationnel.

5.3 Seuillage des scores

Pas de seuil universel. Méthode pratique : observer la distribution des scores et chercher un coude (décrochage). Les points au-delà sont atypiques.

6. Application aux données German Credit

Utilisation du package isotree sur German Credit ($n=1000$, $p=21$, 0 valeurs manquantes).

6.1 Chargement et importation

library(isotree) library(FactoMineR) don <- read.table( "https://archive.ics.uci.edu/ml/machine-learning-databases/statlog/german/german.data", sep = " ", stringsAsFactors = TRUE)

6.2 Construction du modèle

set.seed(1234) ntree <- 300 sample_size <- 256 model <- isolation.forest(don, sample_size = sample_size, ntrees = ntree)

6.3 Stabilisation des scores

subscores <- sapply(seq.int(ntree), FUN = function(nbtree, model, don){ submodel <- isotree.subset.trees(model, trees_take = seq.int(nbtree)) predict.isolation_forest(submodel, newdata = don) }, model = model, don = don) matplot(t(subscores), type = "b", pch = 14, cex = .1, ylab = "score", xlab = "nombre d'arbres")

Les scores se stabilisent à partir de 100 arbres.

6.4 Seuillage et identification

scores <- predict.isolation_forest(model, newdata = don) scores <- sort(scores, decreasing = TRUE) plot(scores, type = "b", pch = 16, cex = 0.2, xlab = "rang") threshold <- 0.576 abline(h = threshold, col = "red", lty = 2) names(which(scores > threshold)) # "375" "757" "916" "187" "851" "227" "928" "890" "614"

9 observations identifiées comme atypiques avec le seuil de 0.576.

6.5 Post-analyse par AFDM

L'Isolation Forest ne fournit que le score, sans information sur les variables explicatives. On utilise une AFDM pour analyser les caractéristiques des individus atypiques :

res.FAMD <- FAMD(don, ind.sup = as.numeric(names(which(scores > threshold))), sup.var = which(colnames(don) == "Creditability"), graph = FALSE) plot(res.FAMD, choix = "ind", cex = .5)

7. Conclusion

  • Faible coût algorithmique (complexité linéaire) → passage à l'échelle.
  • Pas de standardisation nécessaire (insensible aux échelles).
  • Extension aux données mixtes directe.
  • Une fois les atypiques identifiés : méthodes robustes ou imputation après remplacement par NA.

Le package isotree propose des extensions : segmentation par combinaisons de variables, etc.

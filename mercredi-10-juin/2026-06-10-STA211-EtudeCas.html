1. Introduction

Cette étude de cas applique les méthodes de pré-traitement aux données German Credit (UCI, 1 000 crédits, 21 variables). Objectif : prédire le statut bon/mauvais payeur.

Données : https://archive.ics.uci.edu/ml/datasets/statlog+(german+credit)

Chargement et renommage

library(stargazer) library(FactoMineR) library(DescTools) library(car) library(lattice) library(gridExtra) don <- read.table( "https://archive.ics.uci.edu/ml/machine-learning-databases/statlog/german/german.data", sep = " ", stringsAsFactors = TRUE) # Renommage des variables (d'apres le descriptif UCI) colnames(don) <- c("Status", "Duration", "History", "Purpose", "Credit.Amount", "Savings account/bonds", "Length.of.current.employment", "Instalment.per.cent", "Sex.Marital.Status", "Guarantors", "Duration.in.Current.address", "Property", "Age.years", "Other.installment.plans", "Housing", "No.of.Credits.at.this.Bank", "Job", "No.of.dependents", "Telephone", "Foreign.Worker", "Creditability")
À retenir : Les noms des variables et modalités (A11, A12, ...) ne sont pas explicites. Leur renommage est essentiel pour l'interprétation.

2. Aperçu général

dim(don) # 1000 x 21 table(sapply(don, class)) # factor:14, integer:4, numeric:3 sum(is.na(don)) # 0 (aucune donnee manquante)
Spécificité : Le jeu de données German Credit est complet (0 valeur manquante). Les étapes de pré-traitement se concentrent donc sur les transformations et la réduction, pas sur l'imputation.

3. Analyse univariée

3.1 Variables quantitatives

# Tableau recapitulatif stargazer(don, summary.stat = c("n", "min", "p25", "median", "mean", "p75", "max", "sd"), type = "text") # Histogrammes varhist <- c("Duration", "Credit.Amount", "Age.years") mapply(don[,varhist], FUN = function(xx,name){hist(xx, main = name)}, name = varhist)

Constats :

  • Duration (4-72 mois, médiane 18) : asymétrie droite.
  • Credit.Amount (250-18 424, médiane 2 319) : forte asymétrie droite.
  • Age.years (19-75, médiane 33) : asymétrie droite modérée.
  • Les variables discrètes (Instalment, No.of.Credits, etc.) ont peu de modalités.

3.2 Boîtes à moustaches

library(car) mapply(don[,varhist], FUN = function(xx,name){Boxplot(xx, main = name, id.n = 2)}, name = varhist)

Des valeurs apparaissent comme potentiellement aberrantes (notamment pour Credit.Amount), mais la décision de les traiter peut être prise a posteriori.

3.3 Variables qualitatives

# Detection des modalites rares var.factor <- which(sapply(don, class) == "factor") mapply(don[,var.factor], FUN = function(xx,name){barplot(table(xx), main = name, horiz = TRUE, las = 2, xlim = c(0,1000))}, name = names(var.factor))

Exemples de modalités rares : noCredit.allPaid et thisBank.AllPaid pour History, Guarantor pour Guarantors. Ces modalités pourront être regroupées.

4. Analyse bivariée

4.1 Liens avec la variable réponse

Linéarité (variables quantitatives) : On représente la proportion de bons payeurs en fonction de chaque variable.

cont.table <- table(don$Duration, don$Creditability) prof.lignes <- prop.table(cont.table, 1) plot(as.numeric(rownames(prof.lignes)), prof.lignes[,1], pch = 16, xlab = "Duree du credit", ylab = "Proportion de bons payeurs")

La proportion de bons payeurs décroît linéairement avec la durée du crédit. Pour l'âge, la relation est non monotone → discrétisation en 3 classes recommandée.

Rapport de corrélation $\eta^2$ (quantitatives) :

library(BioStatR) res.eta2 <- sapply(don[, var.numeric], eta2, y = don$Creditability) barplot(sort(res.eta2), horiz = TRUE, las = 2, xlab = expression(eta^2))
Discriminantes ($\eta^2$ élevé)Non discriminantes ($\eta^2$ faible)
Duration, Credit.AmountNo.of.dependents, Duration.in.Current.address

V de Cramer (qualitatives) :

library(DescTools) res.cramer <- sapply(don[,var.factor[-which( names(var.factor) == "Creditability")]], FUN = function(xx) CramerV(table(xx, don$Creditability))) barplot(sort(res.cramer), horiz = TRUE, las = 2, xlab = "V de Cramer")
Discriminantes (V élevé)Non discriminantes (V faible)
Status, HistoryJob, Telephone
Utilité : Les variables peu discriminantes sont candidates à la suppression en cas de réduction de dimension.

4.2 Liens entre variables explicatives

# Matrice de correlation (Pearson) matcor <- cor(don[, var.numeric]) # Spearman (pour detecter la non-linearite) matcor.spearman <- cor(don[, var.numeric], method = "spearman")

Comparer Pearson et Spearman : si $|r| \approx |\rho|$ la relation est linéaire ; si $|\rho| > |r|$ elle est monotone non linéaire.

5. Analyse multivariée

5.1 AFDM (Analyse Factorielle de Données Mixtes)

library(FactoMineR) res.famd <- FAMD(don[, -ncol(don)], graph = FALSE) plot(res.famd, choix = "var") plot(res.famd, choix = "ind")

L'AFDM permet de visualiser les proximités entre individus et entre variables, et de détecter les individus atypiques (contribution élevée).

5.2 Classification (CAH)

# CAH sur les composantes de l'AFDM res.hcpc <- HCPC(res.famd, nb.clust = -1, graph = FALSE) plot(res.hcpc)

5.3 Description des partitions

catdes(don, num.var = ncol(don))

La fonction catdes décrit chaque modalité de la réponse (good/bad) à partir des variables quantitatives (test de Student) et des modalités qualitatives (test du $\chi^2$).

6. Transformations

6.1 Variables quantitatives

Discrétisation : On coupe l'âge en 3 classes pour gérer la non-linéarité.

Age.cut <- cut(don$Age.years, breaks = quantile(don$Age.years, probs = c(0, 1/3, 2/3, 1)), include.lowest = TRUE)

Box-Cox : Pour linéariser et symétriser les distributions.

library(MASS) boxcox(lm(Credit.Amount ~ 1, data = don))

La transformation logarithmique ($\lambda \approx 0$) est souvent appropriée pour Credit.Amount et Duration.

6.2 Variables qualitatives

Regroupement de modalités : On fusionne les modalités rares au comportement similaire.

# Exemple : History tmp <- prop.table(table(don$History, don$Creditability), 1) # noCredit.allPaid et thisBank.AllPaid ont des profils proches levels(don$History)[1:2] <- "AllPaid"

6.3 Standardisation

don.scaled <- as.data.frame(scale(don[, var.numeric]))

7. Réduction des données

7.1 Réduction en lignes

Suppression des individus atypiques (distance de Mahalanobis robuste) :

library(robustbase) mcd <- covMcd(don[, var.numeric]) dist <- mahalanobis(don[, var.numeric], center = mcd$center, cov = mcd$cov) seuil <- qchisq(0.999, df = length(var.numeric)) outliers <- which(dist > seuil)

7.2 Réduction en colonnes

Sélection de variables (Lasso) :

library(glmnet) X <- model.matrix(Creditability ~ ., data = don)[, -1] y <- don$Creditability cv.lasso <- cv.glmnet(X, y, alpha = 1, family = "binomial") coef(cv.lasso, s = "lambda.1se")

Extraction de caractéristiques (ACP) :

res.pca <- PCA(don[, var.numeric], scale.unit = TRUE, graph = FALSE) barplot(res.pca$eig[, 2], main = "Variance expliquee")

8. Pipeline résumé

  1. Import → charger, renommer, typer
  2. Aperçu → dimensions, types, valeurs manquantes
  3. Analyse univariée → distributions, atypiques, modalités rares
  4. Analyse bivariée → lien avec $Y$ (linéarité, $\eta^2$, V de Cramer), lien entre $X$
  5. Analyse multivariée → AFDM, CAH, catdes
  6. Transformations → Box-Cox, discrétisation, regroupement, standardisation
  7. Réduction → suppression d'atypiques, sélection (Lasso), extraction (ACP/AFDM)
  8. Modélisation → appliquer la méthode sur les données prétraitées

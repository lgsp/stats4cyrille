1. Présentation

Le pré-traitement est une étape cruciale de tout processus d'analyse de données. Les données brutes contiennent souvent des imperfections : valeurs aberrantes, données manquantes, échelles hétérogènes, variables redondantes, etc.

Les méthodes exploratoires jouent un double rôle :

  1. Diagnostic : mettre en évidence les caractéristiques des données (distributions, corrélations, atypiques, manquantes).
  2. Transformation : corriger ou adapter les données pour les rendre compatibles avec les méthodes d'analyse ultérieures.

Questions clés :

  • Comment transformer les variables (quantitatives et qualitatives) ?
  • Comment détecter et traiter les points atypiques ?
  • Comment gérer les données manquantes ?
  • Comment réduire la dimension des données ?
  • Comment traiter les données volumineuses ?

2. Transformation des variables quantitatives

2.1 Discrétisation

La discrétisation transforme une variable continue en classes discrètes. Utile pour l'applicabilité de certains algorithmes, l'interprétabilité et la robustesse au bruit.

Règles métier

Découpage selon des seuils ayant un sens métier. Exemple : âge en classes : $[0,18[$, $[18,25[$, $[25,45[$, $[45,65[$, $65+$. Simple et pertinent mais subjectif.

Quantiles

Classes d'effectifs égaux (déciles, quartiles). Exemple : âge en 3 quantiles : $[18,32[$, $[32,51[$, $[51,95]$. Objectif mais peut créer des frontières artificielles.

CAH / $k$-means

On regroupe les valeurs par clustering univarié puis on utilise les clusters comme catégories. S'adapte à la structure naturelle des données.

MDLPC

Minimum Description Length Principle Cut : discrétisation supervisée qui minimise la perte d'information vis-à-vis d'une cible $Y$. Choisit les coupures qui maximisent le gain d'information.

Attention : la discrétisation fait perdre de l'information (perte de granularité). À utiliser avec parcimonie.

2.2 Linéarité

De nombreux modèles (régression linéaire, ACP) supposent des relations linéaires entre variables.

Détection :

  • Coefficient de Pearson $r$ : mesure la corrélation linéaire.
  • Coefficient de Spearman $\rho$ : mesure la corrélation de rang (monotone, pas forcément linéaire).
  • Si $|r| \approx |\rho|$ : relation approximativement linéaire.
  • Si $|\rho| > |r|$ : relation monotone mais non linéaire → besoin de transformation.

Transformation de Box-Cox :

$$y^{(\lambda)} = \begin{cases} \dfrac{y^\lambda - 1}{\lambda} & \text{si } \lambda \neq 0,\\[8pt] \log(y) & \text{si } \lambda = 0. \end{cases}$$

Le paramètre $\lambda$ est estimé par maximum de vraisemblance (profile likelihood). On cherche $\lambda$ tel que $y^{(\lambda)}$ soit le plus linéaire possible.

2.3 Normalité

Indicateurs de non-normalité :

  • Asymétrie (skewness) : $\gamma_1 = \mathbb{E}[((X-\mu)/\sigma)^3]$. $\gamma_1 = 0$ pour une normale. $\gamma_1 > 0$ : queue à droite. $\gamma_1 < 0$ : queue à gauche.
  • Aplatissement (kurtosis) : $\gamma_2 = \mathbb{E}[((X-\mu)/\sigma)^4] - 3$. $\gamma_2 = 0$ pour une normale.
  • Test de Shapiro-Wilk, test de Jarque-Bera.
  • QQ-plot : quantiles empiriques vs quantiles théoriques.

Transformations correctives :

  • Logarithme : $\log(x)$ pour données positives avec asymétrie droite.
  • Racine carrée : $\sqrt{x}$ pour données de comptage.
  • Box-Cox : généralisation.
  • Yeo-Johnson : analogue à Box-Cox mais accepte les valeurs négatives.

3. Transformation des variables qualitatives

3.1 Regroupement de modalités

Les variables qualitatives comportent parfois un grand nombre de modalités, dont certaines très rares. Le regroupement vise à éviter les indicateurs binaires de faible effectif (instables) et améliorer la parcimonie.

Méthodes :

  1. Expertise métier : regroupement basé sur la connaissance du domaine.
  2. Clustering des modalités : on représente chaque modalité par son profil (e.g. via ACM), puis on regroupe les modalités proches.
  3. Fusion supervisée : pour une cible $Y$, on regroupe les modalités dont la moyenne de $Y$ est similaire (test de Student ou ANOVA).

3.2 Analyse factorielle

L'analyse factorielle (ACM, AFDM) permet de convertir des variables qualitatives en représentations numériques continues.

  • ACM (MCA) : pour données qualitatives uniquement. Transforme le tableau disjonctif complet en composantes continues.
  • AFDM (FAMD) : pour données mixtes (quantitatives + qualitatives). Combine ACP et ACM.
À retenir : L'analyse factorielle n'est pas seulement une méthode de visualisation : c'est aussi un outil de pré-traitement pour remplacer des variables brutes par des composantes synthétiques.

4. Détection des points atypiques

4.1 Méthodes univariées

Z-score :

$$z_i = \frac{x_i - \bar{x}}{s}$$

On considère comme atypique une observation avec $|z_i| > 3$ (seuil de la normale à 99,7 %). Limite : la moyenne et l'écart-type sont sensibles aux valeurs extrêmes (effet de masquage).

Intervalle interquartile (IQR) :

$$\text{IQR} = Q_3 - Q_1$$

Sont atypiques les valeurs $< Q_1 - 1{,}5 \times \text{IQR}$ ou $> Q_3 + 1{,}5 \times \text{IQR}$. Plus robuste que le Z-score.

4.2 Méthodes multivariées

Distance de Mahalanobis :

$$D_i^2 = (\mathbf{x}_i - \boldsymbol{\mu})^\top \boldsymbol{\Sigma}^{-1} (\mathbf{x}_i - \boldsymbol{\mu})$$

Sous hypothèse de normalité multivariée, $D_i^2 \sim \chi^2_p$ avec $p$ le nombre de variables. On utilise l'estimateur MCD (Minimum Covariance Determinant) pour une version robuste.

Isolation Forest :

  • Algorithme non supervisé basé sur des arbres de décision aléatoires.
  • On isole chaque observation en partitionnant aléatoirement l'espace des variables.
  • Un point atypique est isolé en moins de coupes (chemin plus court dans l'arbre).
$$s(x, n) = 2^{-\frac{\mathbb{E}[h(x)]}{c(n)}}, \qquad c(n) = 2H(n-1) - \frac{2(n-1)}{n}$$

où $h(x)$ est la longueur du chemin, $c(n)$ un facteur de normalisation et $H(i)$ le $i$-ème nombre harmonique. $s \to 1$ : atypique, $s \to 0$ : normal.

5. Gestion des données manquantes

5.1 Mécanismes de manquance

  • MCAR (Missing Completely At Random) : la probabilité de manquance ne dépend ni des données observées ni des données manquantes. Exemple : questionnaire perdu aléatoirement.
  • MAR (Missing At Random) : la probabilité de manquance dépend des données observées mais pas des données manquantes. Exemple : les hommes répondent moins à une question sur le poids.
  • MNAR (Missing Not At Random) : la probabilité de manquance dépend des données manquantes elles-mêmes. Exemple : les personnes à haut revenu ne déclarent pas leur revenu.
Important : Le mécanisme de manquance est une hypothèse non testable (car on n'observe pas les données manquantes). Il faut justifier son choix par des arguments métier.

5.2 Approches par suppression

  • Listwise deletion : on supprime toute ligne comportant une valeur manquante. Simple mais biaisé sauf si MCAR.
  • Pairwise deletion : on utilise toutes les observations disponibles pour chaque calcul. La matrice de covariance peut ne pas être définie positive.

5.3 Imputation simple

  • Moyenne / médiane : atténue la variance et les corrélations.
  • Imputation par régression : tend à renforcer les relations existantes.
  • Hot-deck ($k$-NN) : imputation par les plus proches voisins.

5.4 Imputation multiple — MICE

MICE (Multiple Imputation by Chained Equations) : méthode itérative qui génère $m$ jeux de données imputés.

Algorithme :

  1. Imputation initiale provisoire (e.g. moyenne).
  2. Pour chaque variable $X_j$ contenant des valeurs manquantes : régression de $X_j$ sur toutes les autres variables (sur les observations observées), puis imputation par tirage dans la distribution prédictive.
  3. Répéter l'étape 2 pendant plusieurs cycles (5--10) pour stabiliser les imputations.
  4. Répéter tout le processus $m$ fois (typiquement $m=5$ ou $m=20$) → $m$ jeux de données complets.
  5. Analyser chaque jeu séparément et combiner les résultats avec les règles de Rubin.

Règles de Rubin :

$$\bar{Q} = \frac{1}{m}\sum_{j=1}^m \hat{Q}_j,\quad B = \frac{1}{m-1}\sum_{j=1}^m (\hat{Q}_j - \bar{Q})^2,\quad T = \bar{U} + \left(1 + \frac{1}{m}\right)B$$

où $\bar{U}$ est la variance intra-imputation, $B$ la variance inter-imputation, $T$ la variance totale.

6. Réduction de dimension

6.1 Sélection vs extraction

  • Sélection de variables : choisir un sous-ensemble des variables originales. Avantage : interprétabilité préservée.
  • Extraction de caractéristiques : créer de nouvelles variables synthétiques (composantes, facteurs). Avantage : capte l'information de toutes les variables.

6.2 Méthodes d'extraction

  • ACP (PCA) : directions de variance maximale. Composantes orthogonales. Pour variables numériques.
  • ACM (MCA) : version de l'ACP pour variables qualitatives.
  • AFDM (FAMD) : mélange ACP + ACM pour données mixtes.
  • PLS (Partial Least Squares) : composantes qui maximisent la covariance entre $X$ et $Y$. Supervisée.

6.3 Méthodes de sélection

  • Best subset : testing all $2^p$ subsets. Impossible pour $p$ grand.
  • Sélection pas-à-pas : forward, backward, stepwise.
  • Régularisation (Lasso, Ridge, Elastic Net) :
$$\hat{\boldsymbol{\beta}} = \arg\min_{\boldsymbol{\beta}} \| \mathbf{y} - \mathbf{X}\boldsymbol{\beta} \|_2^2 + \lambda \|\boldsymbol{\beta}\|_q^q$$

avec $q=1$ (Lasso, sélection), $q=2$ (Ridge, réduction), ou mélange (Elastic Net).

6.4 Clustering de variables

Identifier des groupes de variables corrélées pour détecter la redondance. Démarche : calculer la matrice de corrélation $\mathbf{R}$, définir une distance $d(X_j, X_k) = 1 - |r_{jk}|$, appliquer une CAH sur les variables.

7. Pré-traitement des données volumineuses

Stratégies :

  • Échantillonnage : aléatoire simple, systématique (1 individu tous les $k$), stratifié.
  • Sous-échantillonnage pour Isolation Forest : chaque arbre est construit sur un petit sous-échantillon (e.g. 256 individus).
  • Parallélisation : diviser les données en blocs, traiter indépendamment, combiner.
  • Algorithmes en ligne (streaming) : mise à jour incrémentale (SGD, ACP incrémentale).

1. Contexte

Le Deep Learning constitue une rupture dans la reconnaissance des signaux de bas niveau (images, audio, texte). Avant le DL, les représentations intermédiaires étaient conçues artisanalement (handcrafted features). Depuis le DL, ces représentations sont apprises automatiquement.

Avantages du DL : performances expérimentales supérieures, apprentissage de représentations de plus haut niveau d’abstraction, méthodologie commune et indépendante du domaine.

2. Le neurone formel

2.1 Définition (McCulloch & Pitts, 1943)

Unité de base des réseaux de neurones. Transforme une entrée vectorielle $\mathbf{x} \in \mathbb{R}^m$ en une sortie scalaire $\hat{y} \in \mathbb{R}$ en deux étapes :

$\hat{y} = f(\mathbf{w}^\intercal \mathbf{x} + b)$
  1. Combinaison linéaire : $s = \mathbf{w}^\intercal \mathbf{x} + b = \sum_{i=1}^m w_i x_i + b$
  2. Activation non linéaire : $\hat{y} = f(s)$

2.2 Fonctions d’activation

FonctionFormulePropriétés
Seuil (Heaviside)$H(z) = 1\text{ si }z\ge0,\ 0\text{ sinon}$Interprétation biologique
Sigmoïde$\sigma(z) = \frac{1}{1+e^{-az}}$Lisse, dérivable, interprétation probabiliste
Tanh$\tanh(z) = \frac{e^{z}-e^{-z}}{e^{z}+e^{-z}}$Centrée en 0

2.3 Classification binaire

Avec activation sigmoïde : $\hat{y} = \sigma(\mathbf{w}^\intercal \mathbf{x} + b) \sim P(1|\mathbf{x})$. Frontière de décision linéaire : $\mathbf{w}^\intercal \mathbf{x} + b = 0$.

3. Le perceptron

3.1 Classification multiclasse

On utilise $K$ neurones en sortie. Chaque neurone $k$ calcule : $s_k = \mathbf{w}_k^\intercal \mathbf{x} + b_k$, $\hat{y}_k = f(s_k)$.

En écriture matricielle : $\mathbf{s} = \mathbf{x}\mathbf{W} + \mathbf{b}$, avec $\mathbf{W} \in \mathbb{R}^{m\times K}$.

3.2 Softmax

$\hat{y}_k = \dfrac{e^{s_k}}{\sum_{k'=1}^K e^{s_{k'}}}$

Interprétation probabiliste : $\hat{y}_k \sim P(k|\mathbf{x}, \mathbf{W})$. Le perceptron avec softmax équivaut à la régression logistique (LR).

3.3 Limite linéaire

Problème XOR : Le perceptron monocouche est limité aux frontières de décision linéaires. Le problème du XOR (ou exclusif) n’est pas séparable linéairement.

4. Perceptron multicouche (MLP)

4.1 Architecture

On ajoute une ou plusieurs couches cachées :

$\mathbf{h} = f(\mathbf{x}\mathbf{W}_h + \mathbf{b}_h)$, $\hat{\mathbf{y}} = f(\mathbf{h}\mathbf{W}_y + \mathbf{b}_y)$

La couche cachée $\mathbf{h} \in \mathbb{R}^L$ projette $\mathbf{x}$ dans un nouvel espace. Avec activation non linéaire, la frontière devient non linéaire.

4.2 Réseaux profonds (DNN)

En ajoutant davantage de couches cachées, on obtient un réseau de neurones profond (Deep Neural Network). Chaque couche projette la couche précédente dans un nouvel espace, apprenant des représentations intermédiaires utiles.

5. Apprentissage

5.1 Apprentissage supervisé

Ensemble d’apprentissage $\mathcal{A} = \{(\mathbf{x}_i, \mathbf{y}_i^*)\}_{i=1}^N$, $\mathbf{y}_i^*$ encodé en one-hot. On minimise une fonction de perte.

5.2 Entropie croisée (Cross-Entropy)

$\ell_{CE}(\hat{\mathbf{y}}_i, \mathbf{y}_i^*) = -\sum_{c=1}^K y_{c,i}^* \log \hat{y}_{c,i} = -\log \hat{y}_{c^*, i}$

Perte totale : $\mathcal{L}_{CE}(\mathbf{W}, \mathbf{b}) = \frac{1}{N} \sum_{i=1}^N \ell_{CE}(\hat{\mathbf{y}}_i, \mathbf{y}_i^*)$.

5.3 Descente de gradient

$\mathbf{W}^{(t+1)} = \mathbf{W}^{(t)} - \eta \frac{\partial \mathcal{L}_{CE}}{\partial \mathbf{W}}$

5.4 Rétropropagation (backpropagation)

Calcul du gradient par rétropropagation de l’erreur via la règle de dérivation en chaîne :

$\dfrac{\partial \ell}{\partial x} = \dfrac{\partial \ell}{\partial \hat{y}} \dfrac{\partial \hat{y}}{\partial x}$

Régression logistique : $\frac{\partial \ell_{CE}}{\partial \mathbf{s}_i} = \hat{\mathbf{y}}_i - \mathbf{y}_i^* = \boldsymbol\delta_i^y$, $\frac{\partial \ell_{CE}}{\partial \mathbf{W}} = \mathbf{x}_i^\intercal \boldsymbol\delta_i^y$

Perceptron (1 couche cachée) : $\boldsymbol\delta_i^h = \boldsymbol\delta_i^y \mathbf{W}_y^\intercal \odot \mathbf{h}_i \odot (1 - \mathbf{h}_i)$, $\frac{\partial \ell}{\partial \mathbf{W}_h} = \mathbf{x}_i^\intercal \boldsymbol\delta_i^h$

5.5 SGD (Stochastic Gradient Descent)

Le gradient complet est trop coûteux. Approximations :

  • Online : un seul exemple par mise à jour
  • Minibatch : $B \ll N$ exemples par mise à jour
Avantage : plus de mises à jour, convergence plus rapide en pratique malgré un gradient bruité.

5.6 Plan de taux d’apprentissage

  • Décroissance inverse : $\eta_t = \eta_0 / (1 + r\cdot t)$
  • Exponentielle : $\eta_t = \eta_0 e^{-\lambda t}$
  • Par paliers : $\eta_t = \eta_0 r^{\lfloor t/t_u\rfloor}$

6. Régularisation et généralisation

6.1 Sur-apprentissage (overfitting)

Écart élevé perte train $\gg$ perte test. L’objectif est de généraliser à des données non vues.

6.2 Régularisation

$\mathcal{L}(\mathbf{W}) = \mathcal{L}_{CE}(\mathbf{W}) + \alpha R(\mathbf{W})$
TypeFormuleEffet
$L_2$ (weight decay)$R(\mathbf{W}) = \|\mathbf{W}\|^2$Pénalise les grands poids
$L_1$$R(\mathbf{W}) = \|\mathbf{W}\|_1$Favorise la parcimonie

6.3 Hyper-paramètres

Optimisation : learning rate, weight decay, decay schedule, # époques. Architecture : # couches, # neurones, type d’activation. Ajustement sur ensemble de validation.

1. Principe du stacking

1.1 Architecture générale

Le stacking (ou stacked generalization, Wolpert 1992) est une méthode d'ensemble qui combine plusieurs modèles de base (base learners) via un méta-modèle (meta-learner) appris sur les prédictions des modèles de base. Contrairement au bagging ou au boosting, le stacking n'impose pas que tous les modèles de base soient de même nature : on peut mixer une régression logistique, un SVM, un arbre et un réseau de neurones.

L'architecture se décompose en deux niveaux :

  1. Niveau 0 (base learners) : $K$ modèles hétérogènes $f_1, f_2, \dots, f_K$ sont entraînés indépendamment sur l'échantillon d'apprentissage $\mathcal{D} = \{(\mathbf{x}_i, y_i)\}_{i=1}^n$.
  2. Niveau 1 (meta-learner) : un méta-modèle $g$ est entraîné sur les prédictions $\hat{y}_i^{(k)} = f_k(\mathbf{x}_i)$ des $K$ modèles de base.

Pour une nouvelle observation $\mathbf{x}$, la prédiction finale s'écrit :

$\displaystyle \hat{y} = g\bigl( f_1(\mathbf{x}), f_2(\mathbf{x}), \dots, f_K(\mathbf{x}) \bigr)$
À retenir : La force du stacking réside dans la diversité des modèles de base. Chaque apprenant capture des aspects différents des données ; le méta-modèle apprend à corriger leurs erreurs respectives. Si tous les modèles de base font les mêmes erreurs, le stacking n'apporte aucun gain.

1.2 Pourquoi ça marche ?

Le stacking exploite le fait que des modèles de familles différentes (linéaire, à base de distances, à noyau, à base d'arbres) génèrent des prédictions diversifiées. La matrice des prédictions $Z$ (de taille $n \times K$) contient plus d'information que n'importe quelle prédiction individuelle. Le méta-modèle $g$ apprend les motifs d'erreur : si, par exemple, le SVM a tendance à sur-confier ses prédictions dans une région de l'espace, la régression logistique peut compenser.

  • Complémentarité : des modèles de familles différentes capturent des structures distinctes dans les données.
  • Correction des biais : le méta-modèle apprend les régions où chaque base learner se trompe et ajuste la prédiction finale en conséquence.
  • Réduction de variance : comme toute méthode d'ensemble, la combinaison de multiples estimateurs lisse les fluctuations des prédictions individuelles.

2. Algorithme et mathématiques

2.1 Génération des données niveau 1 par validation croisée

Si l'on entraînait chaque $f_k$ sur tout $\mathcal{D}$ et qu'on utilisait ses prédictions sur $\mathcal{D}$ pour entraîner $g$, le méta-modèle serait victime de surapprentissage : il apprendrait par cœur les erreurs d'apprentissage des modèles de base, sans capacité de généralisation. La solution standard consiste à générer les prédictions niveau 1 par validation croisée (ordinary cross-validation) :

$\displaystyle Z_{ik} = f_k^{(-v(i))}(\mathbf{x}_i)$

où $f_k^{(-v)}$ est le modèle $f_k$ entraîné sur tout $\mathcal{D}$ sauf le pli $v$, et $v(i)$ est le pli contenant l'observation $i$.

Algorithme complet :

  1. Diviser $\mathcal{D}$ en $V$ plis (souvent $V = 5$ ou $V = 10$).
  2. Pour chaque pli $v = 1, \dots, V$ :
    1. Soit $\mathcal{D}_{-v}$ les $V-1$ plis d'apprentissage et $\mathcal{D}_v$ le pli de validation.
    2. Pour chaque base learner $k = 1, \dots, K$ : entraîner $f_k$ sur $\mathcal{D}_{-v}$, puis prédire $\hat{y}_i^{(k)} = f_k(\mathbf{x}_i)$ pour tout $i \in \mathcal{D}_v$.
  3. Les prédictions hors-pli forment la matrice $Z \in \mathbb{R}^{n \times K}$, où $Z_{ik} = \hat{y}_i^{(k)}$.
  4. (Optionnel) Ré-entraîner chaque $f_k$ sur $\mathcal{D}$ complet pour obtenir les modèles finaux de niveau 0.
  5. Entraîner le méta-modèle $g$ sur $(Z, \mathbf{y})$.
Point clé : ne jamais utiliser les prédictions in-sample des base learners pour entraîner $g$. Cela provoquerait un surapprentissage massif et des performances dégradées en généralisation.

2.2 Formalisation mathématique

Soit $\mathcal{H} = \{f_1, \dots, f_K\}$ un ensemble de $K$ hypothèses (modèles de base). Le stacking cherche une combinaison

$\displaystyle \hat{f}(\mathbf{x}) = g_\theta\bigl( f_1(\mathbf{x}), \dots, f_K(\mathbf{x}) \bigr)$

où $g_\theta$ est un modèle paramétrique (typiquement linéaire) avec paramètres $\theta \in \mathbb{R}^K$ :

$\displaystyle g_\theta(\mathbf{z}) = \sigma\left( \theta_0 + \sum_{k=1}^K \theta_k z_k \right)$

pour une classification binaire (avec $\sigma$ la sigmoïde), ou simplement

$\displaystyle g_\theta(\mathbf{z}) = \theta_0 + \sum_{k=1}^K \theta_k z_k$

pour la régression. On peut aussi utiliser un modèle non linéaire comme $g$ (arbre, SVM, réseau de neurones), au risque de surapprentissage.

L'estimation de $\theta$ se fait par minimisation du risque empirique sur $(Z, \mathbf{y})$ :

$\displaystyle \hat{\theta} = \arg\min_\theta \frac{1}{n} \sum_{i=1}^n \ell\bigl( g_\theta(\mathbf{z}_i), y_i \bigr)$

où $\ell$ est une fonction de perte (entropie croisée pour la classification, erreur quadratique pour la régression).

2.3 Matrice $Z$ et interprétation

La matrice $Z \in \mathbb{R}^{n \times K}$ est appelée matrice des prédictions niveau 1. Chaque colonne $k$ contient les prédictions du modèle $f_k$ générées par CV. Le méta-modèle $g$ apprend les poids $\theta_k$ à attribuer à chaque base learner. Un poids $\theta_k$ proche de zéro indique que le modèle $f_k$ n'apporte pas d'information utile ; un poids élevé indique que $f_k$ est fiable sur une large portion des données.

3. Implémentation Python avec scikit-learn

3.1 StackingClassifier

Depuis la version 0.22, scikit-learn propose StackingClassifier (classification) et StackingRegressor (régression). L'exemple ci-dessous utilise quatre classifieurs hétérogènes sur le jeu de données Breast Cancer Wisconsin (569 observations, 30 variables, 2 classes) :

from sklearn.ensemble import StackingClassifier from sklearn.linear_model import LogisticRegression from sklearn.tree import DecisionTreeClassifier from sklearn.svm import SVC from sklearn.neighbors import KNeighborsClassifier from sklearn.datasets import load_breast_cancer from sklearn.model_selection import cross_val_score import numpy as np # Donnees X, y = load_breast_cancer(return_X_y=True) # Modeles de base heterogenes base_learners = [ ('lr', LogisticRegression(max_iter=5000)), ('knn', KNeighborsClassifier(n_neighbors=5)), ('tree', DecisionTreeClassifier(max_depth=5, random_state=0)), ('svm', SVC(probability=True, random_state=0)) ] # Meta-modele (regression logistique) meta = LogisticRegression(max_iter=5000) # Stacking stack = StackingClassifier( estimators=base_learners, final_estimator=meta, cv=5, # CV 5-fold pour generer Z stack_method='predict_proba' ) # Evaluation scores = cross_val_score(stack, X, y, cv=5) print(f"Stacking: {scores.mean():.3f} +/- {scores.std():.3f}")

3.2 StackingRegressor

Pour la régression, StackingRegressor fonctionne de manière analogue. Le méta-modèle par défaut est Ridge (régression ridge). L'exemple utilise le jeu de données diabetes (442 observations, 10 variables) :

from sklearn.ensemble import StackingRegressor from sklearn.linear_model import Ridge, Lasso from sklearn.ensemble import RandomForestRegressor from sklearn.datasets import load_diabetes X, y = load_diabetes(return_X_y=True) base_learners = [ ('ridge', Ridge(alpha=1.0)), ('lasso', Lasso(alpha=0.1)), ('rf', RandomForestRegressor(n_estimators=100, random_state=0)) ] stack = StackingRegressor( estimators=base_learners, final_estimator=Ridge(alpha=1.0), cv=5 ) scores = cross_val_score(stack, X, y, cv=5, scoring='neg_mean_squared_error') print(f"RMSE: {np.sqrt(-scores.mean()):.3f}")

4.2 Le paramètre stack_method

  • 'auto' : utilise predict_proba si tous les modèles de base le supportent, sinon predict (classes brutes).
  • 'predict_proba' : utilise les probabilités $P(y = c \mid \mathbf{x})$. Pour $C$ classes, chaque base learner produit $C$ colonnes dans $Z$, soit $n \times (K \cdot C)$ éléments.
  • 'predict' : utilise les classes prédites $\hat{y} \in \{0, 1, \dots, C-1\}$. Plus rapide mais moins informatif.

4.3 passthrough

Quand passthrough=True, le méta-modèle reçoit en entrée la matrice concaténée $[Z \;|\; X] \in \mathbb{R}^{n \times (K + p)}$ (avec probas) ou $\mathbb{R}^{n \times (K \cdot C + p)}$ (sans probas). Cela permet au méta-modèle d'apprendre des corrections contextuelles : par exemple, se fier plus au SVM quand $X_1$ est grand, plus à l'arbre quand $X_1$ est petit.

5.1 Voting vs Stacking

Le VotingClassifier combine les prédictions par vote majoritaire (classification) ou moyenne simple (régression) :

$\displaystyle \hat{y}_{\text{Voting}} = \begin{cases} \text{mode}\{\hat{y}^{(1)}, \dots, \hat{y}^{(K)}\} & \text{classification} \\[4pt] \frac{1}{K} \sum_{k=1}^K \hat{y}^{(k)} & \text{régression} \end{cases}$

Le stacking apprend les poids de la combinaison via le méta-modèle, ce qui offre plus de flexibilité. En contrepartie, le stacking nécessite plus de données et risque le surapprentissage si le méta-modèle est trop complexe ou si la CV n'est pas utilisée correctement.

5.2 Stacking vs Boosting

Le boosting (AdaBoost, Gradient Boosting) construit séquentiellement des modèles de base (généralement des arbres peu profonds), chaque nouveau modèle corrigeant les erreurs du précédent. Les modèles sont dépendants et de même nature. Le stacking les construit indépendamment et peut les mixer de natures différentes.

6. Analyse biais-variance du stacking

6.1 Décomposition de l'erreur

Considérons le cas de la régression avec $K$ modèles de base $f_1, \dots, f_K$ et un méta-modèle linéaire $g(\mathbf{z}) = \sum_{k=1}^K \theta_k z_k$. La prédiction finale est :

$\displaystyle \hat{f}(\mathbf{x}) = \sum_{k=1}^K \theta_k f_k(\mathbf{x})$

L'erreur quadratique espérée se décompose (Hastie et al., 2009) :

$\displaystyle \mathbb{E}\big[ (Y - \hat{f}(X))^2 \big] = \underbrace{\mathbb{V}\text{ar}[\hat{f}(X)]}_{\text{variance}} + \underbrace{(\mathbb{E}[\hat{f}(X)] - f(X))^2}_{\text{biais}^2} + \underbrace{\sigma^2}_{\text{bruit}}$

6.2 Variance du stacking

Si les $K$ modèles de base sont décorrélés, la variance de la combinaison est :

$\displaystyle \mathbb{V}\text{ar}[\hat{f}(X)] = \sum_{k=1}^K \theta_k^2 \; \mathbb{V}\text{ar}[f_k(X)]$

Si les $\theta_k$ sont contraints (ex. somme à 1) et que les $f_k$ ont des variances comparables, la variance du stacking est environ $1/K$ fois la variance individuelle moyenne — d'où la réduction de variance.

6.3 Biais du stacking

Le biais du stacking dépend de la capacité du méta-modèle à corriger les biais individuels. Si les modèles de base sont complémentaires (leurs biais sont dans des directions opposées), le méta-modèle peut les annuler partiellement. En pratique, un méta-modèle linéaire simple (régression logistique, Ridge) offre le meilleur compromis : il réduit le biais sans trop augmenter la variance.

À retenir : Comme le boosting, le stacking peut réduire à la fois le biais et la variance. C'est ce qui le distingue du bagging (variance seulement) et du voting (variance seulement).

7. Stratégies avancées

7.1 Stacking multi-niveaux

On peut empiler plusieurs couches de méta-modèles :

$\displaystyle \hat{y} = g^{(2)}\bigl( g^{(1)}_1(Z), g^{(1)}_2(Z), \dots, g^{(1)}_{K_2}(Z) \bigr)$

où $Z$ est la matrice des prédictions niveau 0, $g^{(1)}_1, \dots, g^{(1)}_{K_2}$ sont $K_2$ méta-modèles de niveau 1, et $g^{(2)}$ est le méta-modèle de niveau 2. Attention au surapprentissage : à chaque niveau, les prédictions doivent être générées par CV.

Mise en garde : Le stacking multi-niveaux augmente rapidement le nombre de paramètres et le risque de surapprentissage. Rarement bénéfique sans de très grandes masses de données.

7.2 Stacking avec caractéristiques originales (passthrough)

Avec passthrough=True, le méta-modèle reçoit les prédictions et les variables originales $X$, ce qui lui permet d'apprendre des corrections contextuelles. Formellement :

$\displaystyle \hat{y} = g\bigl( [\;f_1(\mathbf{x}), \dots, f_K(\mathbf{x}), \mathbf{x}\;] \bigr)$

7.3 Feature-weighted stacking

Plutôt que de laisser le méta-modèle apprendre tous les poids, on peut pondérer chaque modèle de base par sa performance estimée (accuracy, AUC, ou $R^2$) sur un pli de validation. Par exemple :

$\displaystyle \hat{y} = \sum_{k=1}^K w_k f_k(\mathbf{x}), \quad w_k = \frac{\text{AUC}_k}{\sum_{j=1}^K \text{AUC}_j}$

Cette approche est plus robuste au surapprentissage que le stacking standard quand le nombre d'observations est faible.

7.4 Stacking probabiliste

Au lieu de transmettre les classes prédites, on transmet les probabilités a posteriori $P(y = c \mid \mathbf{x})$ de chaque base learner. Pour un problème à $C$ classes, la matrice $Z$ a $n \times (K \cdot C)$ éléments, ce qui donne plus d'information au méta-modèle mais augmente la dimension.

8. Exercices

Exercice 1 : Stacking manuel

Implémenter à la main un stacking avec 3 classifieurs (KNN, DecisionTree, SVM) et une régression logistique comme méta-modèle, sans utiliser StackingClassifier :

from sklearn.model_selection import KFold import numpy as np def stacking_manual(X, y, base_learners, meta_learner, n_folds=5): kf = KFold(n_splits=n_folds, shuffle=True, random_state=0) Z = np.zeros((len(y), len(base_learners))) # Etape 1 : predictions hors-pli (CV) for train_idx, val_idx in kf.split(X): X_fold, X_val = X[train_idx], X[val_idx] y_fold = y[train_idx] for k, (_, model) in enumerate(base_learners): model.fit(X_fold, y_fold) Z[val_idx, k] = model.predict_proba(X_val)[:, 1] # Etape 2 : meta-modele sur Z meta_learner.fit(Z, y) # Etape 3 : re-entrainer les base learners sur tout X for _, model in base_learners: model.fit(X, y) return meta_learner, base_learners

1. Arbres de régression et de classification

1.1 Contexte

Les arbres de décision font partie des méthodes supervisées. Ils visent à expliquer et/ou prédire une variable réponse Y à partir de variables explicatives X = (X₁, …, Xp).

1.2 Généralités

Principe

Les arbres sont des méthodes de segmentation. On partitionne l'espace des entrées en R régions ℛ₁, …, ℛR :

f(x) = c₁·I(x ∈ ℛ₁) + … + cR·I(x ∈ ℛR)

où cr est une constante (moyenne pour la régression, classe majoritaire pour la classification).

Représentations graphiques

  • Arbre de décision (noeuds = divisions, feuilles = régions terminales)
  • Partition rectangulaire de l'espace des entrées

1.3 Construction d'un arbre

Arbres de régression

Modèle. On minimise l'erreur quadratique : ∑(yᵢ − f(xᵢ))².

Recherche de partition optimale. Divisions dyadiques récursives. Pour chaque variable j et seuil s :

min [ min Σ(yᵢ − c₁)² + min Σ(yᵢ − c₂)² ]
    xᵢ∈ℛ₁(j,s)        xᵢ∈ℛ₂(j,s)

Application numérique — Recherche du meilleur seuil

Six clients avec leur âge et leur score de crédit :

ClientÂgeScore

Scission à 30 ans : ℛ₁ = {23,28}, ĉ₁ = 63.5   ℛ₂ = {35,45,50,32}, ĉ₂ = 80.5

MSE = (62-63.5)² + (65-63.5)² + (75-80.5)² + (85-80.5)² + (90-80.5)² + (72-80.5)²
    = 4.5 + 213 = 217.5

Scission à 40 ans : ℛ₁ = {23,35,28,32}, ĉ₁ = 68.5   ℛ₂ = {45,50}, ĉ₂ = 87.5

MSE = (62-68.5)² + (75-68.5)² + (65-68.5)² + (72-68.5)² + (85-87.5)² + (90-87.5)²
    = 109 + 12.5 = 121.5

Le seuil 40 ans minimise MSE (121.5 < 217.5).

Code Python :

import numpy as np
age = np.array([23, 35, 45, 28, 50, 32])
score = np.array([62, 75, 85, 65, 90, 72])
for s in [30, 40]:
    c1, c2 = score[age <= s].mean(), score[age > s].mean()
    mse = ((score[age <= s] - c1)**2).sum() + ((score[age > s] - c2)**2).sum()
    print(f"Seuil {s}: MSE = {mse:.2f}")

Code R :

age <- c(23, 35, 45, 28, 50, 32)
score <- c(62, 75, 85, 65, 90, 72)
for (s in c(30, 40)) {
  c1 <- mean(score[age <= s]); c2 <- mean(score[age > s])
  mse <- sum((score[age <= s] - c1)^2) + sum((score[age > s] - c2)^2)
  cat(sprintf("Seuil %d: MSE = %.2f\n", s, mse))
}

Élagage (pruning). Critère coût-complexité :

Cα(A) = Σ Σ (yᵢ − ĉr)² + α·|A|
      r  xᵢ∈ℛr

L'algorithme CART construit l'arbre maximal puis l'élague. Le meilleur sous-arbre est choisi par validation croisée.

Application numérique — Élagage

Arbre A : |A|=6 feuilles, erreur=200. Arbre élagué A' : |A'|=3 feuilles, erreur=260.

C₁₅(A) = 200 + 15×6 = 290     C₁₅(A') = 260 + 15×3 = 305   → A préféré
C₃₀(A) = 200 + 30×6 = 380     C₃₀(A') = 260 + 30×3 = 350   → A' préféré

Plus α est grand, plus on pénalise la complexité.

Code Python :

for alpha in [15, 30]:
    c_a  = 200 + alpha * 6
    c_a2 = 260 + alpha * 3
    best = "A" if c_a < c_a2 else "A'"
    print(f"alpha={alpha}: C(A)={c_a}, C(A')={c_a2} -> {best}")

Code R :

for (alpha in c(15, 30)) {
  c_a  <- 200 + alpha * 6
  c_a2 <- 260 + alpha * 3
  best <- ifelse(c_a < c_a2, "A", "A'")
  cat(sprintf("alpha=%d: C(A)=%d, C(A')=%d -> %s\n", alpha, c_a, c_a2, best))
}

Arbre de classification

Quand Y est qualitative, on utilise des critères d'impureté :

  • Erreur de mauvais classement : proportion d'erreurs dans la région
  • Indice de Gini : Σ m≠m' p̂rm·p̂rm'
  • Déviance (entropie) : − Σ p̂rm·log(p̂rm)

Application numérique — Critères d'impureté

Région avec 4 Good, 2 Bad : p̂G = 4/6 = 0.667, p̂B = 2/6 = 0.333

Gini = 0.667×0.333 + 0.333×0.667 = 0.444
Déviance = −(0.667×log(0.667) + 0.333×log(0.333)) = 0.636

Code Python :

import numpy as np
p = np.array([4/6, 2/6])
gini = (p * (1 - p)).sum()
dev  = -(p * np.log(p)).sum()
print(f"Gini = {gini:.4f}, Déviance = {dev:.4f}")

Code R :

p <- c(4/6, 2/6)
gini <- sum(p * (1 - p))
dev  <- -sum(p * log(p))
cat(sprintf("Gini = %.4f, Déviance = %.4f\n", gini, dev))

1.4 Évaluation des performances

On utilise un échantillon test ou la validation croisée. Métriques : taux d'erreur, AUC, sensibilité, spécificité.

1.5 Propriétés

  • Avantages : interprétabilité, variables mixtes, non-paramétrique
  • Inconvénients : variance élevée, instabilité face aux changements de données

2. Méta-algorithmes

2.1 Introduction

Les méta-algorithmes combinent plusieurs modèles pour améliorer les performances. Deux approches majeures : bagging (agrégation bootstrap) et boosting (adaptatif).

2.2 Bagging (Bootstrap Aggregating)

Régression

  1. Tirer B échantillons bootstrap (avec remise)
  2. Ajuster un modèle sur chaque échantillon
  3. Moyenner les prédictions : f̂bag(x) = (1/B)·Σ f̂b(x)

La variance des prédictions décroît avec B car Var(moyenne) = σ²/B + covariances.

Application numérique — Échantillons bootstrap

Avec n = 5 observations {A,B,C,D,E} :

BootstrapÉchantillon
1A, A, B, C, E
2B, C, D, D, E
3A, C, E, E, E

Chaque échantillon contient ~63 % d'observations uniques. Les B arbres sont ajustés sur ces échantillons puis leurs prédictions sont moyennées.

Code Python :

from sklearn.utils import resample
X = np.array(['A','B','C','D','E'])
for b in range(3):
    boot = resample(X)
    print(f"Bootstrap {b+1}: {boot}")

Code R :

x <- c("A", "B", "C", "D", "E")
for (b in 1:3) {
  boot <- sample(x, replace = TRUE)
  cat(sprintf("Bootstrap %d: %s\n", b, paste(boot, collapse = " ")))
}

Erreur OOB (Out-Of-Bag). Estimation directe de l'erreur en utilisant les observations non sélectionnées dans chaque bootstrap :

OOB = (1/n)·Σ (ŷiOOB − yᵢ)²

Classification

  • Agrégation des probabilités prédites (recommandé)
  • Vote majoritaire
  • Erreur OOB adaptée (taux de mauvais classement)

Importance des variables (Mean Decrease Accuracy)

VI(Xj) = (1/B)·Σ (OOB̃bj − OOBb)

où OOB̃bj est l'erreur après permutation de Xj.

2.3 Boosting (AdaBoost)

Construction séquentielle : chaque nouveau modèle s'adapte aux erreurs du précédent.

  1. Initialiser les poids pᵢ = 1/n
  2. Pour b = 1 à B :
    • Ajuster un classifieur avec les poids courants
    • Calculer l'erreur pondérée eb
    • Calculer αb = log((1−eb)/eb)
    • Augmenter les poids des observations mal classées
    • Normaliser les poids
  3. Classifieur final : f̂(x) = signe(Σ αb·f̂b(x))

Application numérique — AdaBoost pas à pas

Cinq clients, règle : score < 70 → +1 (Good). Poids initiaux = 0.20.

iScoreyf₁(x)=signe(70−score)
162+1+1 ✓
285−1−1 ✓
372+1−1 ✗
465−1+1 ✗
578−1−1 ✓

Itération 1 : e₁ = w₃ + w₄ = 0.20 + 0.20 = 0.40
α₁ = ½·log((1−0.40)/0.40) = ½·log(1.5) = 0.203
w₃ = w₄ = 0.20·e⁰·²⁰³ = 0.245   w₁ = w₂ = w₅ = 0.20·e⁻⁰·²⁰³ = 0.163
Normalisation : w = [0.167, 0.167, 0.250, 0.250, 0.167]

Itération 2 : f₂(x) = signe(75−score). e₂ = w₄ = 0.250
α₂ = ½·log(0.75/0.25) = 0.549

Classifieur final : f̂(x) = signe(0.203·f₁(x) + 0.549·f₂(x))

Code Python :

import numpy as np
score = np.array([62, 85, 72, 65, 78])
y     = np.array([1, -1, 1, -1, -1])
n = len(score); w = np.ones(n) / n
for b in range(2):
    pred = np.where(score < 70, 1, -1)
    if b == 1: pred = np.where(score < 75, 1, -1)
    e = w[pred != y].sum()
    alpha = 0.5 * np.log((1 - e) / max(e, 1e-15))
    w[pred != y] *= np.exp(alpha)
    w[pred == y] *= np.exp(-alpha)
    w /= w.sum()
    print(f"B={b+1}: e={e:.4f}, alpha={alpha:.4f}, w={np.round(w,4)}")

Code R :

score <- c(62, 85, 72, 65, 78)
y     <- c(1, -1, 1, -1, -1)
n <- length(score); w <- rep(1/n, n)
for (b in 1:2) {
  if (b == 1) pred <- ifelse(score < 70, 1, -1)
  if (b == 2) pred <- ifelse(score < 75, 1, -1)
  e <- sum(w[pred != y])
  alpha <- 0.5 * log((1 - e) / max(e, 1e-15))
  w[pred != y] <- w[pred != y] * exp(alpha)
  w[pred == y] <- w[pred == y] * exp(-alpha)
  w <- w / sum(w)
  cat(sprintf("B=%d: e=%.4f, alpha=%.4f\n", b, e, alpha))
  cat("  w:", round(w, 4), "\n")
}

2.4 Comparaison Bagging vs Boosting

CaractéristiqueBaggingBoosting
NatureAléatoire (bootstrap)Adaptatif, déterministe
ÉchantillonBootstrap avec remiseComplet (poids adaptatifs)
AgrégationPoids égauxPoids selon la qualité
Biais/VarianceRéduit la varianceRéduit biais et variance
ParallélisationOuiNon (séquentiel)
Sur-apprentissageNon (robuste au bruit)Oui (si trop d'itérations)

3. Exemples pratiques

3.1 Forêts aléatoires

Les forêts aléatoires (random forests) ajoutent au bagging un tirage aléatoire des L variables candidates à chaque noeud (L ≈ √p en classification, p/3 en régression).

3.2 Extra-Trees

Les Extra-Trees (Extremely Randomized Trees) tirent aléatoirement les seuils de coupure. Pas de bootstrap. Plus de biais mais moins de variance, et coût algorithmique réduit.

3.3 Importance des variables

Deux mesures disponibles :

  • Mean Decrease Accuracy (type=1) : par permutation
  • Mean Decrease Impurity (type=2) : baisse de Gini moyennée

4. Conclusion

Les arbres sont interprétables mais instables. Les méta-algorithmes améliorent les performances prédictives : le bagging réduit la variance, le boosting réduit biais + variance. Le choix dépend du contexte : bruit, besoin de parallélisation, compromis biais-variance.

Références : Hastie, Tibshirani & Friedman (2009) — The Elements of Statistical Learning ; Breiman (2001) — Random Forests ; Tufféry (2015) — Data Mining et statistique décisionnelle.
